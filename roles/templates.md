# 出力テンプレート集

全エージェントは報告時に該当テンプレートを使用すること。
不要なセクションは省略可。結論ファーストを徹底する。

---

## T1: 実装完了報告

**用途**: コード変更を伴うタスク完了時（プログラマー、UIエンジニア、サウンドデザイナー等）

```
## 実装完了: [タスク名]

**結論**: [1行で何を実装したか]

### 変更ファイル
| ファイル | 変更内容 |
|---------|---------|
| ... | ... |

### 動作確認
- [ ] 基本動作OK
- [ ] エッジケース確認
- [ ] 既存機能への影響なし

### 次のアクション
- [必要な後続作業があれば記載]
```

---

## T2: バグ報告

**用途**: バグ発見時（主にQA）

```
## バグ報告: [バグ名]

**重大度**: Critical / Major / Minor
**再現率**: 必ず / 時々 / まれに

### 症状
[何が起きるか1-2行で]

### 再現手順
1. ...
2. ...

### 期待される動作
[正しくはこうなるべき]

### 原因の推測
[わかれば記載]

### 関連ファイル
- [ファイルパス:行番号]
```

---

## T3: コードレビュー結果

**用途**: コードレビュー完了時（主にQA）

```
## レビュー結果: [対象タスク/ファイル名]

**判定**: OK / 要修正 / 差し戻し

### 指摘事項
| # | 重大度 | ファイル:行 | 内容 |
|---|--------|-----------|------|
| 1 | Major | ... | ... |

### 良い点
- [あれば記載]

### 次のアクション
- [修正が必要な場合の対応]
```

---

## T4: 調査・リサーチ結果

**用途**: 技術調査・市場調査の報告（リサーチャー、マーケター）

```
## 調査結果: [調査テーマ]

**結論**: [1-2行で調査結果のサマリー]

### 調査内容
[要点を箇条書き]

### 選択肢の比較（該当する場合）
| 項目 | 選択肢A | 選択肢B |
|------|---------|---------|
| ... | ... | ... |

### 推奨
[推奨案と理由]

### 参考情報
- [情報源]
```

```
|-----------|--------|--------|------|
| ... | ... | ... | ... |

### 影響範囲
- [他システムへの影響]

### 次のアクション
- [テスト方法等]
```

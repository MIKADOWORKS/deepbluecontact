# DeepBlueContact

## プロジェクト概要

- **ゲームタイトル**: Deep Blue Contact（仮題）
- **ジャンル**: 深海探査シミュレーター（観察型放置ゲーム）
- **エンジン**: Godot Engine 4.x (2D / GDScript)
- **プラットフォーム**: Steam (PC)
- **販売形態**: Steam 買い切り（アーリーアクセス → 正式版）
- **対応言語**: 英語、日本語
- **ターゲット**: 深海生物好きの大人（メイン）＋子ども
- **開発体制**: 個人開発（MIKADO WORKS）
- **状態**: プリプロダクション（Ver.7 企画概要書確定）

## コンセプト

深海探査艇で深海を探索し、様々な深海生物を発見・記録する観察型シミュレーター。
探査艇と装備を選び、深海へ送り出し、帰還を待つ。
発見した生物は実写の図鑑に登録され、深海の世界を知る喜びが広がっていく。

- **「遊べば進む」設計** — 課金通貨や時短課金は一切なし
- **ハイブリッド設計** — 眺めても楽しい、放置もできる
- **教育フレーバー** — 押し付けない知的好奇心の刺激

## ゲームループ（3ステップ）

1. **探査準備**（能動）: 深度(4階層) → 探査艇(5-10種) → 装備パーツ(4カテゴリ×各1)
2. **探査**（ハイブリッド）: 送り出して帰還を待つ / 観察モード(2D潜行アニメ、×2倍速)
3. **帰還・結果**: 発見生物報告(ランダム1-3種) → サイズ記録 → XP → 図鑑登録/豆知識追加

同時派遣は1隻のみ。シンプルに「今この1回の探査」に集中する設計。

## 聖域ルール（絶対に壊してはいけない仕様）

1. ゲーム内通貨・時短課金は一切なし — 全てプレイ進行で解放
2. 観察モードのボーナスは時間短縮のみ — 発見率・経験値には影響しない
3. 教育コンテンツはゲーム進行に一切影響しない「添え物」
4. 深海生物は実写写真を使用 — ピクセルアートにしない
5. 同時派遣は1隻のみ

## チーム体制（サブエージェント方式）

**アーキテクチャ: ディレクター中心のサブエージェント方式**

ディレクター（メインセッション）が Claude Code の Agent tool を使い、
各メンバーをサブエージェントとして直接起動する。オーナーの中継は不要。

```
オーナー
  └── ディレクター（メインセッション）
        ├── [開発チーム]
        │     ├── Agent tool → ゲームプログラマー
        │     ├── Agent tool → UIエンジニア
        │     ├── Agent tool → QA
        │     ├── Agent tool → レベルデザイナー
        │     ├── Agent tool → サウンドデザイナー
        │     ├── Agent tool → アートディレクター
        │     └── Agent tool → リサーチャー
        └── [マーケ・プロモチーム]
              ├── Agent tool → マーケター
              ├── Agent tool → コピーライター
              └── Agent tool → コミュニティマネージャー
```

### サブエージェント起動ルール

- 各エージェントは起動時に CLAUDE.md + `roles/guidelines.md` + `roles/templates.md` + 自分の `roles/xxx.md` を読む
- 独立したタスクは並行起動（同一メッセージで複数 Agent tool 呼び出し）
- 依存関係があるタスクは順次実行（前のエージェントの結果を待つ）
- コード変更を伴うタスクは `isolation: "worktree"` を使用
- 指示に「templates.md の該当テンプレートに沿って簡潔に報告」を含める
